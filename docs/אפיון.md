## 1. מבוא וחזון

### 1.1 תיאור כללי
יישום לניווט שטח (off-road / hiking) בישראל, המשלב **ניווט טופוגרפי**, **תכנון מסלולים**, **הצעת נקודות עניין אטרקטיביות לאורך הדרך**, ו**חוויית טיול קבוצתית בזמן אמת**. היישום פונה למטיילים, רוכבי שטח, נהגי ג'יפים ומשפחות, ומספק חלופה פתוחה ועשירה ליישומים קיימים כדוגמת "עמוד ענן".

### 1.2 הבחנה תחרותית (הייחוד)
מחקר השוק העלה שהשוק הישראלי מפוצל: "עמוד ענן" חזק במאגר POI קהילתי, "Israel Hiking Map" חזק במפה הפתוחה, ויישומים בינלאומיים (Komoot) חזקים בהצעת עצירות ושיתוף. **אף יישום ישראלי אינו משלב באופן אוטומטי:**

> **ניתוב-שטח-ליעד + הצעת עצירות אטרקטיביות מותאמות לאורך המסלול + חוויית טיול קבוצתי בזמן אמת.**

זהו לב הערך המוסף של היישום.

### 1.3 רכיב מרכזי
רכיב המפה הוא המרכז של היישום — כל הפונקציונליות נשענת עליו ומוצגת מעליו.

## 3. עקרונות מנחים

עקרונות שהוגדרו על ידי בעל המוצר ומחייבים לאורך כל הפיתוח:

1. **מקורות מידע חינמיים** — העדפה מובהקת למקורות פתוחים (OpenStreetMap, SRTM, data.gov.il) על פני מקורות בתשלום או סגורים.
2. **מתן קרדיטים** — כל מקור מידע ומפה יקבל ייחוס (attribution) נראה לעין, בהתאם לדרישות הרישיון שלו.
3. **עיצוב חדשני ומזמין** — ממשק מודרני, נקי, ידידותי, שמזמין לשימוש; לא "כלי טכני" יבש.
4. **אופליין-first** — היישום חייב לתפקד גם ללא קליטה סלולרית (שטח פתוח, מדבר, גליל).
5. **פרטיות כברירת מחדל** — שיתוף מיקום ונתונים אישיים הם opt-in ונשלטים על ידי המשתמש.

### 5.2 רכיבי הליבה

| שכבה | טכנולוגיה | תפקיד |
|---|---|---|
| **רינדור מפה** | MapLibre GL (JS לווב, Native למובייל) | רכיב המפה המשותף (R1) |
| **אריחים** | PMTiles (וקטורי, אופליין) + שכבת סאטלייט | מפות רקע (R8), עבודה אופליין |
| **ניתוב שרת** | GraphHopper — פרופיל `hike` | חישוב מסלול שטח (R2) |
| **ניתוב אופליין** | BRouter — אריחי rd5 ישראל | ניתוב ללא קליטה (R2) |
| **backend** | Supabase (קוד פתוח, self-hostable) | Auth (R9), נתונים, זמן-אמת (R10), אחסון (R5), הרשאות |
| **מסד נתונים** | PostgreSQL + PostGIS | נתונים גאוגרפיים, corridor search (R3) |
| **זמן-אמת** | Supabase Realtime (WebSocket) | צ'אט ומיקום חי (R10c, R10d) |
| **מובייל** | React Native | אפליקציית השטח |
| **ווב** | React | ממשק תכנון |

### 5.3 הערה על בחירת Supabase
נבחר משום שהוא מספק את כל הצרכים החדשים (Auth, Realtime, Storage, RLS) בחבילה אחת, הוא **קוד פתוח וניתן ל-self-host** (תואם לרוח הפרויקט), ומבוסס PostgreSQL+PostGIS — בדיוק מה שכבר נדרש ל-POI ולחיפוש המסדרון. חלופה: Firebase (קל יותר אך סגור ופחות מתאים ל-PostGIS).

### 5.4 שיקול מובייל+ווב
React Native (מובייל) + React (ווב) עם MapLibre בשני הצדדים → שיתוף קוד מקסימלי בלוגיקה ובמודל הנתונים. Flutter היא חלופה תקפה.

> ⚠️ כל הקואורדינטות במקורות הישראליים הרשמיים הן ב-**ITM / EPSG:2039** (לא Web Mercator) — נדרש reprojection.

| שכבה | מקור | רישוי | החלטה |
|---|---|---|---|
| **טופוגרפי 1:50K** | **רינדור עצמי מ-OSM** בסגנון טופוגרפי | ODbL (פתוח) | ✅ הדרך הנקייה — חוקי ומלא |

| אורתופוטו מפ"י (רזולוציה גבוהה) | מפ"י | © רישיון בתשלום | ⚠️ רק אם נדרשת רזולוציה גבוהה רשמית |

### 6.2 שבילים ונתוני בסיס

| נתון | מקור | גישה | רישוי |
|---|---|---|---|
| שבילים מסומנים + שביל ישראל | OpenStreetMap | route relations (`osmc:symbol`, `colour`); שביל ישראל = relation 282071 | ODbL |

| רפרנס מומלץ | Israel Hiking Map | קוד פתוח, מבוסס OSM | לבחון repo VectorMap |

### 6.3 נקודות עניין (POI) — לדרישה R3

| סוג POI | מקור | גישה | רישוי |
|---|---|---|---|
| עתיקות | OSM (`historic=archaeological_site`) + GOVMAP `?lay=214090` | Overpass / GOVMAP | ODbL / תצוגה |
| מעיינות | OSM (`natural=spring`) + רשות המים | Overpass / data.gov.il (CSV) | ODbL / פתוח |
| מערות | OSM (`natural=cave_entrance`) | Overpass | ODbL (אין שכבה רשמית) |
| תצפיות | OSM (`tourism=viewpoint`) | Overpass | ODbL |

### 6.4 שירותים ו-APIs
- **Overpass API** — שליפת POI ושבילים (`overpass-api.de/api/interpreter`). לפרודקשן: self-host או חילוץ מ-Geofabrik Israel.
- **GraphHopper / BRouter** — ניתוב.
- **data.gov.il (CKAN)** — מים/תחבורה/קרקע. שים לב: נתוני עתיקות/גנים לאומיים **אינם** שם — הם ב-GOVMAP/OSM.

## 8. מודל נתונים

### 8.1 ישויות מרכזיות (סקיצה)

```sql
-- פרופיל משתמש (כולל אווטאר — R11)
profiles (
  id            uuid PK,        -- מקושר ל-auth.users
  display_name  text,
  avatar_color  text,           -- צבע נבחר
  avatar_icon   text,           -- 'car' | 'jeep' | 'animal_*' ...
  created_at    timestamptz
)

-- טיול (R10b)
trips (
  id            uuid PK,
  owner_id      uuid FK -> profiles,
  name          text,
  planned_route geometry,       -- המסלול המתוכנן (PostGIS)
  status        text,           -- 'planning' | 'active' | 'finished'
  created_at    timestamptz
)

-- חברות בטיול (שולט בהרשאות)
trip_members (
  trip_id       uuid FK -> trips,
  user_id       uuid FK -> profiles,
  role          text,           -- 'owner' | 'member'
  joined_at     timestamptz,
  PRIMARY KEY (trip_id, user_id)
)

-- הזמנות (R9, R10a)
invitations (
  id            uuid PK,
  trip_id       uuid FK -> trips NULL,  -- NULL = הזמנה ליישום בלבד
  inviter_id    uuid FK -> profiles,
  token         text UNIQUE,
  email         text NULL,
  status        text,           -- 'pending' | 'accepted' | 'expired'
  created_at    timestamptz
)

-- צ'אט טיול (R10c) — RLS לפי trip_members
messages (
  id            uuid PK,
  trip_id       uuid FK -> trips,
  user_id       uuid FK -> profiles,
  body          text,
  created_at    timestamptz
)

-- מיקום בזמן אמת (R10d) — ephemeral
live_positions (
  trip_id       uuid FK -> trips,
  user_id       uuid FK -> profiles,
  location      geometry(Point),
  heading       real,
  updated_at    timestamptz,
  PRIMARY KEY (trip_id, user_id)
)

-- מסלולים מוקלטים (R4)
tracks (
  id            uuid PK,
  user_id       uuid FK -> profiles,
  trip_id       uuid FK -> trips NULL,
  geometry      geometry(LineString),
  recorded_at   timestamptz
)

-- תמונות גאו-מתויגות (R5)
photos (
  id            uuid PK,
  user_id       uuid FK -> profiles,
  trip_id       uuid FK -> trips NULL,
  storage_path  text,           -- Supabase Storage
  location      geometry(Point),
  taken_at      timestamptz
)
```

### 8.2 חיפוש עצירות במסלול (R3)
שאילתת corridor search ב-PostGIS:
```sql
-- POI בתוך מסדרון של 1 ק"מ מקו המסלול
SELECT p.* FROM pois p
WHERE ST_DWithin(p.location::geography, :route_line::geography, 1000)
ORDER BY ST_Distance(p.location::geography, :route_line::geography);
```
שלב מתקדם: סינון לפי מרחק-מסלול בפועל (detour) ולא רק מרחק אווירי.

## 9. אבטחה ופרטיות

| נושא | החלטה |
|---|---|
| **הרשאות צ'אט ומיקום** | Row-Level Security ב-PostgreSQL: גישה מותרת רק אם `auth.uid()` קיים ב-`trip_members` של אותו טיול. נאכף ב-DB, לא בקוד הלקוח |
| **שיתוף מיקום** | opt-in בלבד; המשתמש מדליק/מכבה בכל רגע; רק בטיול פעיל |
| **מיקום חי = ephemeral** | נשמר מיקום אחרון בלבד; נמחק בסיום הטיול (להבדיל מ-track שנשמר) |
| **טוקני הזמנה** | חד-פעמיים, עם תפוגה |
| **תמונות** | הרשאות storage לפי בעלות/חברות בטיול |

## 10. קווים מנחים לעיצוב (עיקרון 3 — חדשני ומזמין)

### 10.1 שפה עיצובית
- **מודרני ונקי** — מפה מלאת-מסך, ממשק צף (floating) מעל המפה, מינימום הסחות.
- **מזמין** — צבעוניות חמה הקשורה לטבע ולשטח; אנימציות עדינות; אייקונוגרפיה ידידותית.
- **RTL-first** — עברית מהיסוד.
- **מצב יום/לילה** — קריטי לשטח (מצב לילה חוסך סוללה ונוח בשטח חשוך).

### 10.2 עקרונות UX לשטח
- **כפתורים גדולים** — שימוש בכפפות / בתנועה / בשמש חזקה.
- **קריאוּת בשמש** — ניגודיות גבוהה, גופן ברור.
- **חיווי מצב ברור** — קליטה/אופליין, מצב הקלטה, שיתוף מיקום פעיל.
- **חיווי קבוצתי** — אווטארים צבעוניים (R11) של חברי הטיול על המפה, ברורים ומובחנים.

### 10.3 אווטארים (R11)
ספריית אייקונים: רכב, ג'יפ, ומבחר אייקוני חיות; כל אחד בצבע נבחר. ייצוג עקבי על המפה, בצ'אט וברשימת המשתתפים.
